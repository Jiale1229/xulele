GitHub ssh 密钥配置，常用Git命令

GitHub ssh 密钥配置，常用Git命令

🙆‍♂️ 徐乐乐 🕰︎ 2022/7/21 🏷 git

什么是 `Git`

Git 是目前世界上最先进的分布式版本控制系统！！！

# Git 流程图

图中有四个空间，是 Git 工作流程的精髓所在，分别是：

Remote： 远程仓库，即你在 Github 或者 Gitee 等平台上创建的项目仓库；

Repository： 本地仓库，你可以认为就是我们拉取项目后生成的 .git 文件夹；

Index： 暂存区，事实上它只是一个文件，即 .git 文件夹里面的 index 文件，它保存即将提交到本地仓库的文件列表信息；

workspace： 工作区，即你在 VS code 或者 WebStorm 编译器正在编写的代码。

# 基本配置（初始化）

Git的设置文件为.gitconfig，它可以在用户主目录下（全局配置 ），也可以在项目目录下（项目配置 ）。

# 1.配置用户名和邮箱

# 显示当前的Git配置
git config --list

# 设置提交代码时的用户信息
git config --global user.name '你的名字'
git config --global user.email '你的邮箱'

# 2.创建密钥

# 生成的密钥默认存放在~/.ssh目录下 全部回车即可
ssh-keygen -t rsa -C "xxx@qq.com"

id_rsa：私钥，保存于本地；

id_rsa.pub：公钥，需要把里面内容配置到远端仓库

# 3.在GitHub账户中添加公钥

GitHub–> Settings–> SSH and GPG keys --> New SSH key。

# 4.验证是否配置成功

#验证命令
ssh -T git@github.com

#输入 yes 显示如下说明成功

'Hi XXXXX!  You‘ve successfully authenticated, but GitHub does not provide shell access.'

# 5.完整步骤

# 忽略文件.gitignore

忽略某些文件: $ 创建.gitignore文件，文件中写入需要忽略的文件名（示例:node_modules /dist .idea ...）

# 此行为注释 会被Git忽略

# 忽略 node_modules/ 目录下所有的文件
node_modules


# 忽略所有.vscode结尾的文件
.vscode

# 忽略所有.md结尾的文件
*.md

# 但README.md 除外
!README.md

# 会忽略 doc/something.txt 但不会忽略doc/images/arch.txt
doc/*.txt

# 忽略 doc/ 目录下所有扩展名为txt文件

doc/**/*.txt

如果需要忽略的文件已经提交到仓库，需要删除后，再次提交.gitignore文件才可生效

git rm -r --cached .
git add .
git commit -m 'update .gitignore'
git push -u origin master

# 日常操作命令

# 1.初始化git版本库

git init

# 2.添加/删除文件

# 将当前目录及其子目录下所有变更都加入到暂存区
git add .

# 将仓库内所有变更都加入到暂存区
git add -A

# 添加单个文件到暂存区
git add test.text

# 添加某个后缀的文件到暂存区
git add *.文件后缀

# 添加每个变化前，都会要求确认
# 对于同一个文件的多处变化，可以实现分次提交
git add -p

# 删除工作区文件，并且将这次删除放入暂存区
git rm [file1] [file2] ...

# 停止追踪指定文件，但该文件会保留在工作区
git rm --cached [file]

# 改名文件，并且将这个改名放入暂存区
git mv [file-original] [file-renamed]

# 3.代码提交 commit

# 提交暂存区到仓库区
git commit -m "本次提交的相关说明"
  
# 使用一次新的commit，替代上一次提交
# 如果代码没有任何新变化，则用来改写上一次commit的提交信息
git commit --amend -m "本次提交的相关说明"

Git commit 规范

类型 详细介绍
feat 新特性，添加功能
fix 修改 bug
refactor 代码重构
docs 文档修改
style 代码格式修改, 注意不是 css 修改
test 测试用例修改
chore 其他修改, 比如构建流程, 依赖管理

# 4.推入远程仓库区

# 上传本地分支到远程仓库	
git push origin 分支名

# 强行推送当前分支到远程仓库，即使有冲突
git push [remote] -f

# 清除git仓库的所有提交记录，成为一个新的干净仓库

说明：例如将代码提交到git仓库，将一些敏感信息提交，所以需要删除提交记录以彻底清除提交信息，以得到一个干净的仓库且代码不变

#1.切换到新的分支
git checkout --orphan latest_branch

#2.添加所有文件变化至暂存空间（除了.gitignore中声明排除的）
git add -A
 
#3.提交并添加提交记录
git commit -am "commit message"

#4.删除当前分支 master
git branch -D master

#5.重命名当前分支为 master
git branch -m master

#6.提交到远程master分支 （Finally, force update your repository）
git push -f origin master
